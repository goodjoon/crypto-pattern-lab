# Crypto Pattern Lab — 빌드 및 실행 가이드

## 1. 현재 상태

현재 저장소에는 Gradle 멀티모듈 설정 파일이 들어가 있지만,
아직 Gradle wrapper(`gradlew`)는 추가되지 않았다.

또한 현재 작업 환경에는 `gradle` 명령 자체도 설치되어 있지 않아,
즉시 빌드를 검증할 수는 없는 상태다.

## 2. 다음 작업

아래 중 하나를 수행해야 한다.

### 방법 A — 로컬에 Gradle 설치 후 wrapper 생성
```bash
gradle wrapper
```

### 방법 B — 별도 템플릿이나 초기화 스크립트로 wrapper 추가
프로젝트 표준에 맞는 wrapper 파일을 저장소에 포함한다.

## 3. wrapper 추가 후 권장 확인 절차

```bash
./gradlew tasks
./gradlew build
```

## 4. 왜 wrapper가 필요한가

- 개발 환경 차이를 줄일 수 있다.
- CI/CD 환경에서 일관된 빌드가 가능하다.
- 팀원이 별도 Gradle 설치 없이 시작할 수 있다.

## 5. 현재 기준 결론

지금은 문서/구조/코드 스텁을 먼저 정리한 상태이며,
다음 구현 단계 전에 Gradle wrapper를 저장소에 추가하는 것이 우선이다.
